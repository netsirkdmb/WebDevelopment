Adding Bootstrap to your Project

I am using Bootstrap on my website because it gives me lots of styling tools that allow me to make a website that looks good on both small and large screens. You can find the options for using Bootstrap at this website. I chose to add Bootstrap to my project using a CDN so that I don’t have to host it on my server. To do this, I added the following lines of code to my <head> section in my project html file as described at the above link.



          <!-- Latest compiled and minified CSS -->
          <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.4/css/bootstrap.min.css">

          <!-- Optional theme -->
          <link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.4/css/bootstrap-theme.min.css">

          <!-- Latest compiled and minified JavaScript -->
          <script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.4/js/bootstrap.min.js"></script>

Pros and Cons of a CDN (Content Delivery Network)

Using a CDN means that if someone already has the files for Bootstrap in their browser cache, their computer doesn’t have to download them again and thus, the page can load faster. If you have limited space or bandwidth on your server, then you don’t have to use valuable resources for these files. You can also get the latest version of a CSS file of JavaScript library. The downside of using a CDN is that you are relying on someone else’s server for hosting the files and CDN service that you are using may go down or, depending on your geographic location your CDN service may be blocked.

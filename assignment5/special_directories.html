Special Directories

Meteor automatically includes all CSS and JavaScript files in the project directory eliminating the need to link to them in your HTML file. You should be careful not to add extra CSS and JavaScript files to your project folder since they will always be sent to server and/or client and they will affect the performance of your site. By default, Meteor sends HTML and CSS files to the client, while JavaScript files are sent to both the client and the server. For JavaScript files, you can control which section of code is used with the Meteor.isClient and Meteor.isServer variables. If you want Meteor to load a file differently from the default behavior described above, you must use the following special directories: /client, /server, /public, and /private. I will describe what each directory is used for below.


/client

The /client directory holds any files that you only want sent to the client. In this project, I am using it to store the files for Custombox. Inside, the /client directory, you can also have a /compatibility directory. This directory is used for JavaScript files that use global variables. Since Custombox depends on global variables, some of its files are stored in the /compatibility directory. I will go over adding Custombox to your project later.


/server

Logically, the /server directory is for any files that are only sent to the server. These may include files that have data that you donâ€™t want visible to the client.


/public

The /public directory is used for files that are sent un-modified to the client like images.


/private

The /private directory is only accessible from the server and will not be covered in this project.

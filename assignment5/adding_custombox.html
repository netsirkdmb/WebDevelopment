Adding Custombox to your Project

On my website, I wanted to be able to add another feeding without needing to have the required fields visible all of the time. Because Meteor apps work in real time, they do not allow you to use multiple pages for your app. Therefore, I could not use a button for adding a feeding that would link the user to a separate page with a form for inputting the ata. I decided to solve this problem with Custombox modal windows. That way, when the user clicks the button to add a feeding, a window pops up and allows the user to enter the required information and save it.

Custombox did not have the option to use a CDN for hosting the required files so you must download them and host them on your server. You can download Custombox and learn about it at this website. Instead of including the files as described in the “How to use” section of this website, you need to open up the “dist” directory in the zip file that you downloaded from this website. The “dist” directory contains three files. Copy the custombox.min.css file to the /client directory in your Meteor project folder. Then copy the custombox.min.js and legacy.min.js files to the /compatibility directory that is inside the /client directory in your Meteor project folder. Now you are ready to use Custombox on your Meteor project. Choose which effect you want to use on your modal window (I chose the Fadein effect) and add the following code to the Meteor.isClient section of your JavaScript file. Since we are using Meteor, this code is slightly different from the code that appears on the Custombox website in the “How to use” section.



          Template.body.events({
            "click #add": function (e) {
              Custombox.open({
                target: "#modal-add",
                effect: "fadein"
              });
              e.preventDefault();
            }
          });

Note: I have given my “Add Feeding” button an id of “add” and I have created a div with an id of “modal-add” that contains all of the HTML for the form to add feeding information to my log. In my CSS file, I have given my modal div some styling to make it pretty and I have set its display to none so that it doesn’t show up unless the “Add Feeding” button is clicked and the Custombox modal window appears.
